## Project Architecture Constraints (Bizu SaaS)

Estas regras complementam a constituição e refletem o stack **Vercel-only**
deste repositório. Agentes devem respeitá-las ao planejar e implementar features.

### Deploy e runtime

- **Target exclusivo:** Vercel (React Router preset `@vercel/react-router`).
- **Não** reintroduzir VPS, Docker, Portainer ou `react-router-hono-server`.
- **Não** usar `node build/server/index.js` como modelo de produção.
- Server entry: `src/server.ts` (Web API — Hono + `createRequestHandler`).

### Stack obrigatória

- **Frontend:** React 19 + React Router v7 Framework Mode (`ssr: true`).
- **API:** Hono em `src/api/app.ts`, montado em `/api` via `src/server.ts`.
- **Dados:** Drizzle ORM + Postgres próprio (`DATABASE_URL` com pooler em serverless).
- **Auth/auxiliar:** Supabase (Auth, Storage, Functions, Realtime) — **não** CRUD via `supabase.from()`.

### Convenções de código

- Rotas em `src/routes/`; componentes de página em `src/pages/` (reutilizados pelas routes).
- Dashboard **client-only** — sem `loader` server-side com dados sensíveis.
- Validar APIs e forms com Zod compartilhado.
- Idioma da UI: pt-BR.
